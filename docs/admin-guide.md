# מדריך מנהל מערכת — בדרך טובים

מסמך זה מסביר תפעול שוטף של האתר עבור המנהל הטכני.

## הקצאת תפקידים

לאחר התחברות ראשונה של משתמש, ברירת המחדל היא `reader`.

- כדי להפוך משתמש לכותב — `/admin/users` → בחר משתמש → "כותב".
- אדמין נוסף — קבע ידנית ב-DB או דרך אותו ממשק.

## ניהול קטגוריות

`/admin/categories` — הוספה, מחיקה, וסידור מחדש (חצים למעלה/למטה ושמירה).
ה-slug נוצר אוטומטית כתעתיק לטיני מהשם העברי.

## תור LLM (תקצירים וגרסה מעובדת)

המערכת מריצה עבודות LLM ברקע. ה-Cron של Vercel מוגדר ל-`*/2 * * * *` (כל 2 דקות).

### הפעלה ידנית

מסך `/admin/jobs` מציג סטטוס. כדי להוסיף עבודה למאמר ספציפי:

```bash
curl -X POST https://your-domain.com/api/admin/llm/enqueue \
  -H "content-type: application/json" \
  -H "cookie: <session cookie>" \
  -d '{"articleId": 42, "task": "summary"}'
```

(או יוסף UI מלא בשלב הבא — כפתור "הפק תקציר" / "הפק גרסה מעובדת" בעמוד המאמר ב-/author.)

### פרומפטים

ברירת המחדל בקוד: `lib/llm/index.ts`. כדי לשנות בלי לעדכן קוד — שמור גרסה חדשה בטבלת `llm_prompts` עם `is_active = true`. הריצה תשתמש בגרסה הפעילה האחרונה.

## תגובות מדווחות

`/admin/comments` מציג רק תגובות עם דיווח 1 ומעלה או מוסתרות. המנהל יכול לבדוק ולהחליט.

## סקירת תפעול שוטף

| משימה | תדירות |
|---|---|
| בדיקת `/admin/jobs` לשגיאות | פעם בשבוע |
| בדיקת תגובות מדווחות | פעם ב-2–3 ימים |
| גיבוי DB ידני (לאחר חודש פעילות) | פעם בשבוע — `pg_dump` |
| בדיקת אחוז אחסון R2 | פעם בחודש |
| בדיקת שימוש חינמי של Neon | פעם בחודש |

## משתני סביבה לבדיקה

ראה `.env.example`. אחרי כל שינוי משתנה ב-Vercel — נדרש Redeploy.

## בעיות נפוצות

### האימייל לא מגיע
- ודא ש-`RESEND_API_KEY` תקין.
- ודא ש-`RESEND_FROM_EMAIL` הוא מדומיין שאומת ב-Resend.
- בדוק את הלוגים של Vercel.

### העלאת קובץ נכשלת
- אם הקובץ `.doc` — ודא ש-`CLOUDCONVERT_API_KEY` מוגדר.
- אם הקובץ גדול — Vercel Hobby מוגבל ל-25MB. שקול הקטנה.

### חיפוש לא מחזיר תוצאות צפויות
- ודא ש-`search_vector` מתעדכן (יש לבדוק עם `EXPLAIN` או `SELECT search_vector FROM articles LIMIT 1`).
- אם איכות תוצאות חיפוש בעברית גרועה — ראה תוכנית שדרוג Meilisearch ב-`plan.md`.
